# Nuclei 统一接入任务队列清单

## 目标

将现有 `Nuclei扫描` 从“独立页面即时执行”统一到当前项目的任务队列、检测记录、导出报告体系中，但**不修改** `nuclei.exe` 与模板本身，只改本项目的上层包装层。

## 原则

- 保留现有 `nuclei_service.py` 作为执行器
- 保留单次扫描能力，但新增“任务化扫描”主路径
- Nuclei 与 Python POC 在检测记录中统一展示
- 检测记录、导出报告、状态统计尽量复用现有实现
- 不改 Nuclei 模板格式，不改原始 CLI 行为

## 任务拆分

### 1. 扩展批量任务数据模型

- 为 `batch_tasks` 增加任务来源/任务类型标识
- 为 `batch_task_items` 增加 Nuclei 所需字段
- 明确区分：
  - `poc` 子任务：`url + poc_id`
  - `nuclei` 子任务：`url + template_path`
- 保证旧数据兼容，不影响已有 POC 批量任务

验收标准：

- 原有 POC 批量任务不受影响
- 新任务模型能容纳 Nuclei 任务

### 2. 在批量任务服务中接入 Nuclei 调度

- `batch_task_service.py` 支持创建 Nuclei 任务
- 后台 worker 能识别 `engine_type = nuclei`
- 每个子任务调用 `nuclei_service` 执行
- 结果转换为统一摘要：
  - `命中 / 未命中 / 异常`
  - `reason`
  - `detail_file`

验收标准：

- 后端能创建并跑完一批 Nuclei 子任务
- 检测记录可读取统一摘要

### 3. 增加 Nuclei 任务 API

- 新增创建 Nuclei 扫描任务接口
- 支持：
  - 单 URL + 多模板
  - 单 URL + 文件夹
  - 多 URL + 多模板
- 保留原即时扫描接口，不直接删除

验收标准：

- 前端可通过新 API 创建任务
- 创建后能在检测记录中查询到

### 4. 改造 Nuclei 前端交互

- `Nuclei扫描` 页新增“创建扫描任务”主入口
- 创建任务后自动跳转到 `检测记录`
- 保留必要的模板浏览和选择体验
- 不再把即时返回结果作为唯一主路径

验收标准：

- 从 Nuclei 页创建任务后，用户能在检测记录页继续查看进度与结果

### 5. 统一检测记录展示

- 检测记录页识别 Nuclei 任务
- 任务卡片显示：
  - 任务来源 `POC / Nuclei`
  - 目标数、模板数、命中数、异常数
- 详情页支持展示 Nuclei findings 摘要
- 导出报告兼容 Nuclei 任务

验收标准：

- 用户在一个页面里能同时查看 POC 与 Nuclei 两类检测任务
- 导出结果不报错，内容可读

## 开发顺序

1. 扩展批量任务数据模型
2. 接入 Nuclei 调度与统一摘要
3. 新增 Nuclei 任务 API
4. 改造 Nuclei 前端交互
5. 统一检测记录展示与导出

## 非目标

- 不修改 `nuclei.exe`
- 不修改 Nuclei 模板本身
- 不把现有即时扫描能力全部删掉
- 不引入新的独立任务系统，继续复用现有 `batch_tasks`
