# 结构化失败分类任务清单

## 目标

在不推翻现有执行链路的前提下，为单条执行、批量执行、检测记录提供一套**可演进的最小结构化失败分类**，为后续的：

- 失败原因展示
- 失败统计
- 自动二次修复生成

打基础。

## 设计原则

1. 不追求一次性分完所有失败类型，只做**可扩展的粗分类**。
2. 第一阶段不新建表，优先在现有结果表上增量加字段。
3. 先由系统规则完成第一层分类，不把基础分类交给大模型。
4. 允许保留 `unknown` / `other` 作为兜底。
5. 先覆盖最常见、最稳定、最有价值的失败类型。

## 第一阶段：最小结构化分类落地

### 1. 后端结果模型扩展

在单条执行结果和 `batch_task_items` 中增加：

- `failure_category`
- `failure_code`
- `failure_stage`
- `retryable`

第一阶段仅实现以下大类：

- `not_vulnerable`
- `network_error`
- `code_error`
- `environment_error`
- `oob_error`
- `nuclei_error`
- `unknown`

### 2. 规则分类器

新增统一分类模块，基于明确规则做第一层分类：

- 语法错误 -> `code_error / syntax_error`
- 导入错误/依赖缺失 -> `environment_error / import_error`
- DNS/连接/超时 -> `network_error / xxx`
- OOB 运行环境不可用 -> `oob_error / runtime_unavailable`
- Nuclei 引擎不可用/扫描失败 -> `nuclei_error / engine_unavailable|scan_failed`
- 正常执行但未命中 -> `not_vulnerable / no_evidence_found`
- 其他未知异常 -> `unknown / unclassified`

### 3. 批量记录持久化

在 `batch_task_items` 中保存结构化分类字段，供：

- 检测记录详情
- 导出报告
- 后续统计与修复

使用。

### 4. 前端最小展示

第一阶段只在两个位置展示：

- 单条执行结果弹窗
- 检测记录详情弹窗

列表页先不增加复杂筛选，只展示简化标签：

- 命中
- 未命中
- 异常

详情里再显示：

- 分类
- 代码
- 阶段
- 是否建议重试

### 5. 测试

至少补：

- 分类器单元测试
- 单条执行结果分类测试
- 批量结果落库与读取测试

## 第二阶段：记录页筛选与统计

在检测记录页增加：

- 按 `failure_category` 筛选
- 按 `retryable` 筛选
- 常见错误统计

## 第三阶段：失败修复链路

基于结构化失败分类，补自动二次修复：

- 输入：漏洞信息 + 原始 POC + 执行结果 + 结构化分类
- 输出：是否建议修复 + 修复后的 POC

## 当前执行顺序

1. 第一阶段：后端字段与规则分类
2. 第一阶段：前端最小展示
3. 第一阶段：测试与兼容验证
4. 第二阶段与第三阶段后续再开
